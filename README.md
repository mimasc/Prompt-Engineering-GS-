# Proposta do professor: 

# Nesta GS, vocês deverão utilizar técnicas de Prompt Engineering para projetar um assistente virtual que seja capaz de responder, de maneira clara e empática, a perguntas relacionadas a como reagir em situações de desastres naturais.
# O foco será desenvolver prompts eficazes e estruturados para garantir que o assistente responda de forma contextualizada, considerando as necessidades e emoções de três perfis principais de usuários:
# 1.	Vítimas diretas do desastre, que podem estar buscando instruções imediatas de segurança.
# 2.	Moradores da região afetada, que desejam se informar sobre medidas preventivas e atualizações.
# 3.	Familiares das vítimas, que podem querer orientações sobre como ajudar ou localizar entes queridos.
# Objetivos:
# Aplicar técnicas de prompt engineering, como contexto, few-shot learning, role prompting e structured output, para controlar e refinar as respostas do modelo de linguagem.
# Desenvolver a capacidade de criar prompts sensíveis ao contexto e ao perfil emocional do usuário.
# •	Explorar os desafios éticos e técnicos de projetar assistentes para situações críticas, onde a informação precisa ser precisa, clara e responsável.
# •	Experimentar diferentes estruturas de prompt para testar como elas influenciam a qualidade e adequação das respostas.
# Orientações para Realização:
# 1.	Escolha de Perfis: Cada grupo poderá escolher um ou mais perfis de usuário para focar: vítima, morador ou familiar. Se quiserem, podem também projetar prompts/assistentes diferentes para cada um.
# 2.	Pesquisa e Coleta de Informações: Pesquisem boas práticas de resposta a desastres naturais, como enchentes, terremotos, incêndios ou deslizamentos, para que o assistente forneça informações corretas e úteis.
# 3.	Criação dos Prompts:
# o	Criem exemplos de prompts que garantam que o modelo de linguagem se posicione como um assistente especializado em resposta a desastres.
# o	Testem diferentes estratégias, como fornecer exemplos no próprio prompt (few-shot) ou definir explicitamente o papel(role) do assistente (role prompting).
# 4.	Testes e Ajustes: Utilizem a interface de um modelo de linguagem (como OpenAI GPT) ou outras ferramentas disponíveis para testar os prompts criados e avaliem:
# •	Clareza e utilidade das respostas.
# •	Adequação emocional e sensibilidade.
# •	Correção das informações.
# 5.	Documentação: Produzam um relatório ou apresentação que inclua:
# •	Exemplos dos prompts/códigos utilizados.
# •	As respostas geradas pelo modelo.
# •	Análise crítica sobre os resultados.
# •	Possíveis melhorias. 


# Exemplos de Situações a Simular:
# •	“Estou preso dentro de casa com minha família, a água está subindo, o que devo fazer?”
# •	“Sou morador da cidade X, como me preparo para uma possível evacuação?”
# •	“Minha irmã está desaparecida após o terremoto, como posso tentar localizá-la?” 

# Critérios de Avaliação:
# •	Criatividade e adequação na elaboração dos prompts.
# •	Clareza e qualidade das respostas obtidas.
# •	Reflexão sobre os limites e responsabilidades éticas do uso de modelos de linguagem em contextos de risco.
# •	Qualidade da documentação e apresentação final.
